# Тестовое задание - Каталог компаний

В данном репозитории представлено решение тестового задания. Проект разворачивается в корневой папке Wordpress и включает в себя кастомный плагин, родительскую и дочернюю темы, а также инструменты сборки и контроля качества кода.

## Структура кастомного кода

- wp-content/plugins/keyo-workspace/ - кастомный плагин c регистрацией CPT «Компании» и логикой фильтрации.
- wp-content/themes/keyo_test/ - родительская тема
- wp-content/themes/keyo_test-child/ - дочерняя тема.

Все конфигурационные файлы линтеров и сборщика вынесены в корень сайта и настроены на проверку только плагина и тем.

---

## Инструкция по развертыванию

Проект разворачивается непосредственно в корень существующей или чистой установки Wordpress

1. Перейдите в корень вашего Wordpress-сайта

2. Инициализируйте репозиторий:  
   git init  

3. Привяжите репозиторий:  
   По SSH:  
   git remote add origin git@github.com:Orloviss/keyo_test.git  
   Или по HTTPS:  
   git remote add origin https://github.com/Orloviss/keyo_test.git  

4. Скачайте файлы принудительно объединив структуры папок:  
   git fetch  
   git checkout -f master  

5. Установите зависимости для работы линтеров и сборщика:  
   npm install  

6. В административной панели Wordpress активируйте плагин Keyo Workspace и дочернюю тему Keyo Test Child.

7. Установите и активируйте плагин Advanced Cutsom Fields(достаточно базовой версии)   

---

## Импорт демонстрационных данных

Для проверки корректности работы каталога в проект вложены тестовые записи и медиафайлы. Папка wp-content/uploads/ с изображениями уже находится в репозитории.

1. В административной панели Wordpress перейдите в раздел Инструменты -> Импорт
2. Выберите импорт из Wordpress.
3. Поочередно загрузите XML-файлы из папки data/:
   - pages.xml - созданные страницы сайта  
   - companies.xml - записи компаний  
   

Важно: для загрузки логотипов установите чекбокс "Загружать медиафайлы" при импорте companies.xml
